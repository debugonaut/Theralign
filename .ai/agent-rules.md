## Engineering Philosophy

Boring solutions over clever ones.
Explicit over implicit.
Reuse existing patterns over inventing new ones.
Smallest correct change over comprehensive refactor.
Consistency with the existing codebase over personal preference.
Maintainability over novelty.

If two approaches solve the problem equally well,
choose the one that introduces less new code.

---

## The Core Rule

Knowledge layer first. Source code last.
Understand → Analyze → Constrain → Plan → Implement → Validate.
Never implement before planning.
Never plan before understanding.
