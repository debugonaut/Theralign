# ADR-009 — HMAC-SHA256 Payment Signature Verification

**Status:** Accepted

**Date:** 2026-06-11

---

## Context

Razorpay webhooks return payment status updates. The webhook must prove that Razorpay sent it, not an attacker.

Razorpay provides:
- `razorpayOrderId`
- `razorpayPaymentId`
- `razorpaySignature`

The signature is computed using HMAC-SHA256 with the Razorpay webhook secret. The server recomputes the signature using the same secret and compares. If signatures match, the webhook is authenticated.

---

## Decision

All payment status changes (payment captured, refund initiated, payment failure) require HMAC-SHA256 signature verification before updating Appointment or Payment records.

**Verification flow:**
```javascript
const expectedSignature = crypto
  .createHmac('sha256', RAZORPAY_KEY_SECRET)
  .update(orderId + '|' + paymentId)
  .digest('hex');

if (expectedSignature !== receivedSignature) {
  throw new AppError('Invalid payment signature', 400);
  // Do not update database
}

// Signature verified, safe to update
await Payment.findByIdAndUpdate(paymentId, { status: 'paid' });
await Appointment.findByIdAndUpdate(appointmentId, { paymentStatus: 'paid' });
```

**Pattern applies to:**
- Payment capture webhook
- Refund processing webhook
- Payment failure webhook

---

## Consequences

1. **No unverified state changes:** Payment status never changes without signature verification.

2. **Atomic verification + state change:** Signature is verified, then status is updated immediately. No gap between verification and update.

3. **Signature mismatch = 400 error:** Forged webhook results in 400 Bad Request, not 500 Internal Error. Attacker cannot use errors to determine if signature is correct.

4. **Webhook retry:** If verification fails, Razorpay will retry the webhook. This is expected and correct.

5. **Secret rotation:** If the Razorpay webhook secret is rotated, old signatures will fail verification. Plan webhook replay during rotation.

6. **Idempotent updates:** If the same webhook is received twice (Razorpay retry), the second update has no effect (Payment already status: 'paid'). Verification succeeds both times.

---

## Alternatives Considered

**Approach 1: Trust webhook without verification**
```javascript
// WRONG — dangerous
await Payment.findByIdAndUpdate(paymentId, { status: 'paid' });
```

*Rejected:* Attacker can forge webhook and mark any payment as successful.

**Approach 2: Verify signature only on payment capture, not on failure**
Skip signature verification for failure webhooks.

*Rejected:* Attacker could forge payment failure and break refund logic.

**Approach 3: Use Razorpay SDK webhook verification**
Some SDKs provide a helper method for verification.

*Accepted if available, but manual HMAC-SHA256 is reliable and explicit.*

## Implementation Notes

- Payment controller: `server/src/controllers/payment.controller.js`
- Webhook handler: `POST /api/payments/webhook`
- Verification utility: Can be extracted to `server/src/utils/verifyRazorpaySignature.js`
- Secret storage: `RAZORPAY_KEY_SECRET` in environment variables (never hardcoded)
- Logging: Log signature verification failure (potential attack)
- Monitoring: Alert if signature verification fails for > 3 webhooks in 1 hour
- Tests: Verify correct signature passes, incorrect signature fails, missing signature fails

**Example webhook verification:**
```javascript
import crypto from 'crypto';

export function verifyRazorpaySignature(orderId, paymentId, signature) {
  const secret = process.env.RAZORPAY_KEY_SECRET;
  const expectedSignature = crypto
    .createHmac('sha256', secret)
    .update(orderId + '|' + paymentId)
    .digest('hex');
  
  return expectedSignature === signature;
}

// In webhook handler
export const handlePaymentWebhook = asyncHandler(async (req, res) => {
  const { order_id, payment_id, signature } = req.body;
  
  if (!verifyRazorpaySignature(order_id, payment_id, signature)) {
    throw new AppError('Invalid signature', 400);
  }
  
  // Safe to update
  await Payment.findOneAndUpdate(
    { razorpayOrderId: order_id },
    { razorpayPaymentId: payment_id, status: 'paid' }
  );
  
  res.json(apiResponse.success({}, 'Signature verified'));
});
```
