# ADR-003 — Cloudinary resource_type Configuration for File Types

**Status:** Accepted

**Date:** 2026-06-11

---

## Context

Theralign stores multiple file types in Cloudinary:
- **Credentials:** PDF documents (doctor licenses, certificates)
- **Session notes:** PDF documents (clinical notes after appointment)
- **Avatars:** JPEG, PNG, WebP images (patient/doctor profile pictures)

Cloudinary supports different `resource_type` values:
- `image` — optimized for images (JPG, PNG, WebP)
- `video` — optimized for video (MP4, WebM)
- `raw` — for any file type, no transformation, direct storage
- `auto` — Cloudinary detects file type automatically

The `resource_type` affects:
- Storage location and retrieval speed
- Available transformations and optimizations
- API response schemas
- Deletion behavior

---

## Decision

**For credentials and session notes (PDFs):**
Use `resource_type: 'raw'` when uploading to Cloudinary.

```javascript
const upload = await cloudinary.uploader.upload(filePath, {
  resource_type: 'raw',
  public_id: `credentials/${doctorId}/${filename}`,
  // no transformations — raw PDFs are delivered as-is
});
```

**For avatars (images):**
Use `resource_type: 'auto'` and let Cloudinary detect the image type.

```javascript
const upload = await cloudinary.uploader.upload(filePath, {
  resource_type: 'auto',
  public_id: `avatars/${userId}/${filename}`,
  transformation: [
    { width: 200, height: 200, crop: 'fill' },
    { quality: 'auto' }
  ]
});
```

---

## Consequences

1. **PDFs cannot be transformed:** Cloudinary transformations (resize, crop, format convert) only work on `image` and `video` resource types. PDFs stored as `raw` are immutable.

2. **PDFs use different deletion API:** Raw files require the public_id to specify resource type:
   ```javascript
   await cloudinary.uploader.destroy(publicId, { resource_type: 'raw' });
   ```

3. **Retrieve PDFs as-is:** PDF URLs returned from Cloudinary are direct download links. No transformation suffix needed.

4. **Images are optimized:** Images are automatically optimized for web delivery. Cloudinary may reformat JPEG to WebP if browser supports it.

5. **Storage impact:** PDFs are delivered as uploaded. No size optimization. Important for large credential files (< 100MB typical).

6. **Bandwidth impact:** Raw PDFs may use more bandwidth than transformed PDFs. However, medical credentials should not be transformed (lossy compression not acceptable for legal documents).

---

## Alternatives Considered

**Approach 1: Use resource_type: 'auto' for everything**
Let Cloudinary detect file type and handle appropriately.

*Rejected:* Works, but explicit resource_type is clearer. Auto-detection could theoretically fail for PDFs with unusual extensions.

**Approach 2: Use resource_type: 'image' for PDFs**
Try to force PDFs into image handling.

*Rejected:* PDFs are not images. Cloudinary may fail or deliver unexpected results.

**Approach 3: Store PDFs on separate service (S3, etc.)**
Keep Cloudinary for images only, use S3 for documents.

*Rejected:* Adds infrastructure complexity. Cloudinary handles both well.

## Implementation Notes

- Upload service in `server/src/services/upload.service.js` determines resource_type based on file mimetype
- Doctor credential PDFs: `resource_type: 'raw'`
- Session note PDFs: `resource_type: 'raw'`
- Avatar images: `resource_type: 'auto'`
- Deletion code must pass correct `resource_type` to `cloudinary.uploader.destroy()`
- Tests verify PDF upload as `raw` and image upload as `auto`
